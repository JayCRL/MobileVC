# Optimization Suggestions

Last updated: 2026-04-24

This file tracks remaining improvement areas after the latest reconnect, QR CWD, and default model fixes.

## Already Addressed Recently

- Launcher QR URLs now include `cwd` from the current `mobilevc` invocation directory.
- Flutter scan parsing now applies QR `cwd` to the connection config.
- The launcher passes `RUNTIME_WORKSPACE_ROOT=process.cwd()` to the backend.
- Claude empty/default model now resolves to `default` and launches as plain `claude`.
- Flutter sends application-level WebSocket `ping` events and handles server `pong` events.
- Flutter sends `session_resume` on connect/reconnect/foreground recovery when a session is selected.
- Server WebSocket writes have a deadline to avoid stuck writer goroutines on broken mobile connections.

## 1. Increase Replay Robustness

- `runtimeSession` still uses a bounded pending buffer. Consider increasing the limit or switching to a time/size-based ring buffer.
- Consider persisting critical prompt/interaction events into session projection so replay survives process restarts.
- Add lightweight event sequence diagnostics in debug logs to compare client cursor vs server cursor.

## 2. Split `SessionController` Responsibilities

`SessionController` still owns connection, command dispatch, pending prompts, reviews, files, skill/memory state, runtime info, ADB state, and notifications.

Suggested split:

- Connection/reconnect/resume coordinator.
- Runtime/agent state reducer.
- Review/permission reducer.
- File/diff/terminal state reducers.
- Catalog state for skill/memory.

## 3. Make WebSocket Payloads More Incremental

- Large session history, terminal logs, and diff payloads should support pagination or lazy hydration.
- Timeline/diff/file panels should request only visible data where practical.
- Repeated runtime info and catalog refreshes should use lightweight cache/version metadata.

## 4. Formalize Permission and Review State

- Permission, review, plan, reply, and continue-input states should be modeled as explicit finite states.
- Reduce string-based branching in UI button enabling/disabling logic.
- Keep one source of truth for whether the input bar is locked, waiting for approval, or ready for text.

## 5. Improve Mobile Long-Task UX

- Show a clear ?connection stale / recovering? banner when activity is visible but no server events arrive.
- Surface replay result metadata (`replayedCount`, `latestCursor`, `runtimeAlive`) in debug logs or a debug panel.
- Consider showing ?last update received at ?? in the activity area for long-running tasks.

## 6. Add Focused Tests

Suggested high-value tests:

- Flutter config parsing for QR URLs containing `cwd`.
- Launcher `buildLaunchUrl(...)` includes token and cwd.
- `session_resume` request payload includes cursor/state when selected session exists.
- Server `ping` action emits `pong`.
- Send failures in `MobileVcWsService` emit `ws_send_error` and trigger controller recovery.
