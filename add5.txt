【架构升级指令：从 Web 终端蜕变为 Vibe Coding 智能面板】

兄弟，我们后端的 PTY 引擎非常完美，但目前的业务呈现完全偏离了我们的愿景。
用户现在抱怨：“这只是个命令行可视化，根本没有 Vibe Coding 的感觉！” >
我们要彻底重构前端的体验，从“被动的终端”升级为“主动的 AI 编程助手”。请执行以下前端（web/index.html）和关联后端的升级：

1. 引入 Markdown 渲染与代码高亮 (UI 视觉升级)
我们的底层将要运行 claude code 等 AI CLI，它们会输出大量 Markdown 代码块。

请在 index.html 中通过 CDN 引入 marked.js 和 highlight.js（或等效的轻量级库）。

当收到 type: "log" 且判定为 AI 回复内容时，必须将其渲染为带语法高亮和优雅排版的 HTML 卡片，而不是干瘪的纯文本。代码块右上角必须带一个原生的“一键复制”小图标。

2. 增加“快捷动作盘” (Dynamic Action Panel)
手机打字极度痛苦，必须将常用交互按钮化。

在底部输入框的上方，新增一个横向可滑动的按钮区（Action Panel）。

静态快捷键： 默认放置如【Ctrl+C (中断)】【清屏】等基础操作。

动态快捷键： 当前端收到 type: "prompt_request" 时，除了显示文字提示，还要将 options（如 ["y", "n"]）直接渲染成该区域的醒目大按钮（绿色同意/红色拒绝）。用户点击即发送 {"action": "input", "data": "y\n"}。

3. UI 布局 Chat 化 (交互模式切换)

把原本密密麻麻的终端日志流，改成类似微信/ChatGPT 的气泡对话流样式。

用户的输入（无论是自然语言还是命令）靠右侧显示为气泡；服务端的输出卡片（Logs/Errors/Markdown）靠左侧显示。

4. 【后端可选验证】把底层进程换成真正的 AI CLI

(如果你本地已全局安装了某些 AI 工具)：在客户端默认发送的 exec 命令中，不再是启动 sh，而是允许直接启动带有交互的 CLI 程序，让这套前端真正为 AI 包装一层移动端皮肤。
