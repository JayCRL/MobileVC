# 优化建议

## 1. 统一默认配置与环境注入
- 将 Flutter 端与 Go 后端的 host / port / token / cwd 默认值统一到同一配置源，避免前后端默认端口不一致带来的连接失败。
- 建议把可变配置集中到环境变量 + 本地持久化两层，UI 只负责展示和修改，不再散落在多个入口。

## 2. 收敛 SessionController 的状态职责
- `SessionController` 当前承载了连接、命令发送、等待态、diff 审核、文件浏览、Skill / Memory、Runtime Info、会话恢复等大量逻辑，建议继续拆成更细的状态模块或 reducer。
- 目标是把“状态计算”和“UI 触发”分离，降低后续新增事件类型时的回归成本。

## 3. 强化 WebSocket 事件的增量化
- 对 timeline、diff、terminal log、session history 等高频数据，尽量采用增量 patch 或分页加载，减少一次性推送大对象。
- 对前端来说，优先只同步当前视图需要的字段，历史数据按需拉取。

## 4. 进一步规范权限 / 审核的有限状态机
- 继续把 permission、review、reply、continue input 这些等待态收敛成显式状态机，减少字符串判断和分支漂移。
- 对按钮、输入框、文件查看器的锁定规则做统一约束，避免不同入口出现行为不一致。

## 5. 优化移动端大列表与长时间线渲染
- 对 chat timeline、diff 列表、terminal execution 列表做懒加载和可视区域渲染，避免会话变长后 UI 卡顿。
- 对 Markdown、diff 语法高亮和文件预览做缓存，减少重复构建成本。

## 6. 提升后台通知与前后台切换体验
- 将“是否需要用户介入”的信号进一步去重和节流，减少同一轮状态反复触发通知。
- App 回到前台后可优先刷新当前 active session 的关键状态，避免用户手动重试。

## 7. 完善后端 runner 与 PTY 的可观测性
- 为 PTY 启动、首次输入、权限阻塞、等待用户输入、退出码等关键节点补充更明确的结构化日志。
- 建议把 runner 生命周期事件与 session 投影事件对齐，方便排查“前端看见了什么、后端实际发生了什么”。

## 8. 让文件 / diff / 日志读取更轻量
- 文件查看、diff 审核、终端日志切换都可以优先返回摘要，再按需展开全文。
- 对高频切换内容做本地缓存，降低重复 WebSocket 往返。

## 9. 统一测试金字塔
- 保持 `session_controller` 的单测覆盖等待态、权限决策、review 决策、恢复会话等核心路径。
- 对关键交互补少量集成测试，重点覆盖“后端事件 -> 控制器状态 -> UI 锁定/解锁”的闭环。

## 10. 收敛重复配置与重复入口
- Skill / Memory / 会话恢复等能力已经形成完整闭环，后续优化重点应放在入口一致性、状态同步和性能，而不是继续增加平行入口。
- 对功能面板的展示和操作路径保持单一主入口，减少用户在多个位置重复操作的认知负担。
